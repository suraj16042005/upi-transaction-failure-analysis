# UPI Transaction Failure Analysis Dashboard

## Overview

This project presents a Power BI dashboard designed to analyze UPI transaction failures and identify operational patterns across banks, transaction hours, and failure categories.

The dashboard helps stakeholders understand transaction reliability, peak failure periods, bank-wise performance, and overall system efficiency.

**Note:** This project uses a synthetic dataset created for analytical and portfolio demonstration purposes.

---

## Business Problem

Digital payment systems process millions of transactions every day. Even a small percentage of failures can impact customer experience and business operations.

The objective of this dashboard is to:

* Monitor transaction success and failure rates.
* Identify banks with higher failure volumes.
* Analyze failure reasons.
* Detect peak hours of transaction failures.
* Evaluate transaction response times.

## Analysis Performed

### Bank-wise Failure Analysis

Identifies banks contributing the highest number of failed transactions.

### Failure Reason Distribution

Analyzes the major causes of transaction failures, including:

* Bank Downtime
* Timeout Issues
* Network Issues
* Server Errors

### Hourly Failure Trend

Tracks transaction failures across different hours of the day to identify peak risk periods.

### Response Time Analysis

Compares average response times between successful and failed transactions.

---

## Key Insights

* SBI recorded the highest number of failed transactions among the selected banks.
* Transaction failures increased significantly during evening peak hours.
* The overall transaction success rate remained at 95%.
* Failure reasons were distributed across multiple operational categories, indicating system-wide monitoring requirements.
